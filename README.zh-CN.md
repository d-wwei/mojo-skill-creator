# Mojo Skill Creator

一套造出真正好用的 AI Agent 技能的方法论——不只是结构正确。

## 问题

大多数 Skill 都能通过验证。Frontmatter 格式对，目录干净，触发准确。但产出是扁的。两个不同任务跑出两组词，套在同一副骨架上——一样的深度，一样的节奏，一样的没个性。

差距不在技术。在方法论。现有的 Skill Creator 教你怎么打包一个 Skill。没有一个教你怎么设计一个连领域专家看了都觉得「像回事」的 Skill。

## 核心思路

在写任何红线、任何流程步骤之前，**先去研究这个 Skill 要自动化的那件事，世界上最好的人是怎么做的。**

这不是比喻。是字面意思的第一步：

1. 找到这个 Skill 对应的真实学科
2. 找到公认的顶尖实践者——人、公司、团队
3. 研究他们的真实工作流——会议分享、公开的操作流程、访谈里透露的实战细节（不是营销材料）
4. 提取他们的质量标准——什么会被他们打回去？什么不可妥协？
5. 跨学科扫描——同样的问题结构在别的领域存在吗？

专家的工作流变成 Skill 的工作流。专家的标准变成 Skill 的红线。专家的思维方式变成 Skill 的姿态。

### 实际效果

在改进一个前端设计 Skill 时，这一步挖出来的东西有：Apple 的「Creative Selection」（一个包装盒做了 500 个原型），Linear 的「设计是搜索，不是生产线」，Stripe 的三层质量标准（能用 + 好用 + 精致），Airbnb 把设计稿打印出来并排铺在板上做物理审计。

这些洞察没有一个来自设计原则或 AI Prompt 工程。全部来自研究最好的团队实际怎么干活。结果：重新设计了一套 6 阶段工作流，加了承诺前的探索阶段，用多方向并排对比替代单一「最佳猜测」，以及约束收紧梯度——探索时松，交付时严。

那个 Skill 的特异性测试通过率从 ~20% 升到 ~85%。方法论有效，是因为它导入的是真实专业经验，不是抽象规则。

## 建立在研究之上的两个机制

领域研究提供原材料。两个机制把原材料转化为 Skill 的质量：

### 锁死下限：约束驱动设计

来自 [ljg-skills](https://github.com/lijigang/ljg-skills)——14 个持续产出有性格内容的技能。

**告诉 Agent 不能做什么。**「每句话不得包含超过一个从句」可以检验。「写得清楚一点」不行。从专家质量标准（领域研究中发现的）导出的红线，比凭空设计的红线有效得多。

实践中：
- **红线先于流程。** 5-10 条不可违反的约束，每条配机械检测方法，在写任何内容之前锁死。
- **姿态替代角色。** 认知位置（「不教课，不演讲——先亮自己的弯路，再给方向」）而非身份标签（「你是专家」）。
- **验收标准前置。** 3-5 条可测试的完成条件，基于专家实际使用的质量标杆。

### 抬高上限：知识层级诊断

来自在 66 品牌设计系统上的真实改进实践。

Skill 在三个层级中的某一层运行：

| 层级 | 产出长什么样 | 升级路径 |
|------|------------|---------|
| **原则层** | 正确但千篇一律——所有项目同一个模板 | 添加场景特异的模式 |
| **模式层** | 识别场景，质量稳定，但没辨识度 | 从领域顶尖实践者构建案例资产库 |
| **案例层** | 有清晰身份感，能发现并纠正偏差 | 维护和深化标杆参考 |

每层的升级路径不同。对原则层的 Skill 用案例层的技术是浪费力气。先诊断，再针对性干预。

## 两个命令

**`new`** — 9 步。关键新增：第 2 步是领域最佳实践研究，后面所有步骤都基于它。

```
1. 理解场景 → 2. 研究领域最佳实践 ★ →
3. 设计红线 + 验收标准（基于研究结果）→
4. 规划 Token 架构 → 5. 规划资源 → 6. 构建 Skill →
7. 人类可审计（可选）→ 8. 跨平台验证 → 9. 迭代
```

**`boost`** — 4 阶段。Phase 1.7 是领域最佳实践研究，把 Skill 当前工作流和专家工作流做对比。

```
Phase 1: 诊断（结构 + 知识层级 + 效率 + 领域研究 ★）
Phase 2: 处方（架构→机械→内容，严格顺序）
Phase 3: 执行（按序修复，每类修完验证）
Phase 4: 报告（人类可审计的升级总结）
```

## 工程

方法论是核心。工程保证它落地：

- **跨平台。** Claude Code、Codex、Gemini CLI、OpenClaw。语义化动词，不用平台工具名。
- **Token 高效。** 四层架构——本 Skill 的 SKILL.md 只有 478 词。每次只加载一个子命令的工作流。
- **分发就绪。** 自包含的包。终端用户 symlink 即用。不需要全局安装。
- **可选自进化。** 打包 [skill-se-kit](https://github.com/d-wwei/skill-se-kit)（~48KB），子 Agent 隔离执行运行时学习。

## 快速开始

```bash
git clone https://github.com/d-wwei/mojo-skill-creator.git
ln -sf "$(pwd)/mojo-skill-creator" ~/.claude/skills/mojo-skill-creator
# Codex/Gemini: ln -sf ... ~/.agents/skills/mojo-skill-creator
```

然后说：「创建一个新技能」或「升级这个技能」。
