# Mojo Skill Creator

一套造出真正好用的 AI Agent 技能的方法论——不只是结构正确。

## 大多数 Skill 的问题

大多数 Skill 都能通过验证。Frontmatter 格式对，目录结构干净，description 触发准确。但产出平庸。同一个 Skill 跑两个不同任务，结果换了词，骨架一模一样——一样的节奏，一样的深度，一样的没个性。

差距不在结构。在设计方法论。

现有的 Skill Creator 教你**怎么打包一个 Skill**。没有一个教你**怎么设计一个能产出有辨识度、稳定可靠的 Skill**。能通过验证和真正好用之间，隔着方法论。

## Mojo 带来什么

Mojo 融合两套经过验证的方法论，从相反的方向夹击质量问题：

### 锁死下限：约束驱动设计

来自对 [ljg-skills](https://github.com/lijigang/ljg-skills) 的深度解剖——14 个技能，持续产出有性格的内容，不只是正确的内容。

核心洞察：**告诉 Agent 不能做什么，比告诉它要做什么有效得多。**「每句话不得包含超过一个从句」可以检验——逐句扫描。「写得清楚一点」不可检验——每个 Agent 的理解都不一样。

在实践中：

- **红线先于流程。** 一个 Skill 在定义任何步骤之前，先定义 5-10 条不可违反的约束。每条约束配一个机械检测方法。质量底线在第一个字之前就锁死。
- **姿态替代角色。**「你是一位资深写作教练」产出模仿。「不教课，不演讲，不聊天。先亮自己的弯路，再给方向」产出精确的认知位置。精确度差一个数量级。
- **验收标准前置。** 在写任何东西之前先定义 3-5 条可测试的「完成」条件。测试方法：两个独立评审会不会对通过/不通过产生分歧？会的话，标准太模糊。
- **领域反模式库。** 写作类 Skill 有写作反模式（被动句堆积、AI 腔、翻译腔）。视觉类有视觉反模式（纯黑、Inter 字体、三等分布局）。每条都是二元检查。

### 抬高上限：知识层级方法论

来自真实的技能改进实践——在一个 66 品牌设计系统上系统性地从泛泛提升到精准。

核心洞察：**技能在三个知识层级中的某一层运行，每层之间的升级路径完全不同。**

| 层级 | 产出长什么样 | 怎么诊断 |
|------|------------|---------|
| **原则层** | 正确但千篇一律。所有项目的产出像同一个模板。 | 给两个风格完全不同的任务。如果产出只换了表面内容——结构、节奏、间距一模一样——原则层。 |
| **模式层** | 能识别场景类型，应用对应模式。稳定但没辨识度。 | 要求「按 X 的风格」产出。抓住了大方向但漏掉了具体参数——模式层。 |
| **案例层** | 有清晰的身份感。能发现并纠正偏差。 | 按参考风格产出，故意引入一个错误。如果 Skill 能自动捕获——案例层。 |

从原则层升到模式层，需要添加场景特异的模式。从模式层升到案例层，需要构建案例资产库——标杆参考案例，每个覆盖四个维度（参数、操作指南、品质底线、直觉校验）。每条案例必须通过特异性测试：去掉名字，还能猜出是谁吗？

### 同一个问题的两面

ljg 方法论锁死下限：产出不可能比红线允许的更差。
知识层级方法论抬高上限：产出从泛泛走向精准和有辨识度。

合在一起解决单方面做不到的事——既**可靠**（永远不低于底线），又**好用**（明显超越泛泛）。

## 两个命令

**`new`** — 从零创建。8 步：理解场景 → 设计红线和验收标准（在写任何内容之前）→ 规划 Token 预算 → 识别可复用资源 → 构建 Skill → 可选加入自进化和人类可审计 → 跨平台验证 → 迭代。

**`boost`** — 诊断升级。4 阶段：诊断（结构审计 + 知识层级 + Token 效率 + 跨平台检查）→ 处方（架构→机械→内容，严格顺序）→ 执行 → 报告。

## 工程也管

方法论是核心。但工程也不能掉链子——设计再好的 Skill，只能在一个平台上跑、或者一加载就吃光 context，在实践中还是不行。

- **跨平台：** Claude Code、Codex、Gemini CLI、OpenClaw 都能跑。指令用语义化动词，不用平台工具名。
- **Token 高效：** 四层架构——元数据 → 路由(≤500词) → 工作流(≤2000词) → 参考资料(按需)。本 Skill 自身的 SKILL.md 只有 475 词。
- **分发就绪：** 自包含的包。终端用户 symlink 即用。不需要全局安装、不需要 setup 脚本。
- **可选自进化：** 可打包 [skill-se-kit](https://github.com/d-wwei/skill-se-kit)（~48KB）实现运行时学习。子 Agent 处理反馈提取——主 Skill 零 Token 开销。

## 快速开始

```bash
# 1. Clone
git clone https://github.com/d-wwei/mojo-skill-creator.git

# 2. Symlink 到你的 Agent
ln -sf "$(pwd)/mojo-skill-creator" ~/.claude/skills/mojo-skill-creator
# Codex/Gemini: ln -sf ... ~/.agents/skills/mojo-skill-creator

# 3. 开始用
# 「创建一个新技能」→ 激活 new
# 「升级这个技能」→ 激活 boost
```

## 方法论来源

- **[ljg-skills](https://github.com/lijigang/ljg-skills)** — 14 个锁死下限而非追求上限的技能。约束驱动设计、姿态替代角色、领域反模式库。
- **人类工作流方法论** — 知识层级诊断、案例资产库、约束收紧梯度、缺陷分层修复。来自一个 66 品牌设计系统的真实改进（199 文件，19,209 行，并行 Agent 约 45 分钟完成）。
- **[Anthropic skill-creator](https://github.com/anthropics/claude-code)**（Apache 2.0）— 六步创建流程、渐进式披露、SKILL.md 格式规范。
