## 本次迭代经验

### 做对了
- 4 层 token 架构从设计阶段就规划，避免了后期重构
- 让 skill 自身成为方法论的活示范（architecture-as-demonstration）
- Review 阶段抓到了 token 预估偏差和红线自矛盾问题

### 需改进
- Layer 2 workflow 文件实际词数比 plan 预估少 20-30%，说明 plan 阶段对 Markdown 内容的词数预估偏高
- 应在 plan 阶段就对每个文件做结构大纲，更精确预估

### 新发现
- SKILL.md 里的 token 预估是高杠杆信息——agent 用它做加载决策，错误预估直接影响 context 利用效率
- "两人独立评审一致性"是检验红线可机械性的好 proxy test
